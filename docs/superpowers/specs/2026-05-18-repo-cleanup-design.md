# Repo cleanup — design

**Date:** 2026-05-18
**Branch:** `chore/repo-cleanup` (to be created from `fix/derive-counts-from-data`)
**Scope:** Local-only cleanup of a personal fork of `ambicuity/New-Grad-Jobs`. No remote pushes.

## Goal

Bring the fork to a minimal state that mirrors upstream's working code while removing community-only automation, stale planning docs, and root-level cruft. After this work, syncing with upstream should be straightforward and the local checkout should be navigable.

## Non-goals

- No refactor of `scripts/update_jobs.py` or other working scraper code — tested and not broken.
- No reorganization of `tests/` — current layout is fine.
- No README rewrite — content under `## 📂 Browse` is auto-generated by `scripts/sync_readme_counts.py` and the job-table generator.
- No changes to `docs/` (the static GH Pages site).
- No remote git operations — no `git push`, no `git push --delete`, no branch deletions on `origin`.
- No deeper code refactor (module splits, test reorganization). Out of scope this round.

### Commit 5 — prune local branches

Algorithm:
1. Compute merged set: `git branch --merged main | grep -vE '^\*|main$'`.
2. Compute PR-mirror set: `git branch | grep -E '^\s*(pr-[0-9]+(-(head|merge|review|fix|local|finalize|refresh))?|backup-pr[0-9]+.*)$'`.
3. Union; subtract `main` and current branch (`fix/derive-counts-from-data`).
4. Present the deletion list to the user for one final confirmation before destructive action.
5. Record SHAs in the commit message footer so any branch can be resurrected with `git branch <name> <sha>`.
6. Delete merged branches with `git branch -d`. Force-delete PR-mirror branches with `git branch -D` only after operator confirms each PR exists on the upstream remote (so deletion is non-destructive — the work lives on GitHub).

Keep:
- `main`
- `fix/derive-counts-from-data` (current)
- Any `feat/*` or `fix/*` the operator flags during the confirmation step.

## Verification gate

After every commit:
- `pre-commit run --all-files` passes
- `pytest -q` passes
- `git status` shows only the expected change set

If any check fails, stop and either fix in-place or `git reset --hard` to the previous commit.

## Rollback

- Each commit is independently revertable with `git revert <sha>`.
- Branch deletions in commit 5 record SHAs in the commit message footer so deleted branches can be reconstructed: `git branch <old-name> <sha>`.

## Risk register

| Risk | Mitigation |
|------|------------|
| `pages-deployment.yml` references an output from a deleted workflow | Pre-flight grep before commit 4; either keep the dependency or inline it |
| `update-jobs.yml` chains `auto-merge-scraper.yml` via `workflow_run` | Same pre-flight grep |
| `codecov.yml` references removed `tests.yml` matrix | Prune in commit 4 |
| A kept `feat/*` branch is unmerged work the operator forgot about | Operator-confirmation step in commit 5 |
| `scripts/validate_config.py` rename misses an import | Grep for `test_config` across the tree before committing |
| `.gitignore` rule `/terminal-*.png` is too narrow if files move under subdirs | Acceptable — these are all root-level artifacts |
